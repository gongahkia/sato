## mood-music — Emotion-Driven Music Player (YouTube Music + mpv)

Architecture:
- Go: TUI (Bubble Tea) + mpv IPC client (JSON over Unix socket)
- Python: emotion detection (DeepFace/OpenCV) + music search (ytmusicapi)
- IPC between Go and Python: JSON over stdin/stdout
- Playback: mpv with yt-dlp backend, controlled via JSON IPC over Unix socket
- Config: TOML at `~/.config/mood-music/config.toml`

blockedBy: [22]

**PURPOSE**

3. Implement these methods:
   - `Play(url string) error` -- `{"command": ["loadfile", url, "replace"]}`

   - `GetVolume() (int, error)` -- `{"command": ["get_property", "volume"]}`
   - `Seek(seconds float64) error` -- `{"command": ["seek", seconds, "relative"]}`
   - `GetPosition() (float64, error)` -- `{"command": ["get_property", "time-pos"]}`
   - `GetDuration() (float64, error)` -- `{"command": ["get_property", "duration"]}`
   - `GetTitle() (string, error)` -- `{"command": ["get_property", "media-title"]}`
4. All IPC calls should have a 2-second timeout — WHY: mpv IPC is local Unix socket so responses are near-instant; 2s is generous enough for yt-dlp metadata fetches but catches truly hung connections.

**DONE WHEN**

- [x] Timeout errors are handled gracefully

### Task 26 (A) +mpv

blockedBy: [23, 24]

**PURPOSE**
Pre-fetches next tracks so playback is seamless when the current track ends.

**WHAT TO DO**
1. In Go, create `queue/manager.go` with `QueueManager` struct holding a `[]Track` slice.
2. When a mood is detected/changed, Python searches YouTube Music and returns 10 tracks — WHY: 10 tracks at ~3min avg = ~30min of playback per mood, balancing queue depth against staleness if mood changes.
3. Go stores these in the queue. When mpv goes idle (track ends), immediately play the next track from queue.
4. When queue drops below 3 tracks, request more from Python for the current mood — WHY: 3 tracks = ~9 minutes buffer, ensuring seamless playback; at ~3min per track, this gives enough runway for a YouTube Music search round-trip.
5. On mood change, clear the queue and fetch new tracks for the new mood.
6. Implement `Next()`, `Previous()` (go back in played history), `Current() Track`, `Upcoming() []Track`.

**DONE WHEN**
- [x] Seamless playback between tracks (no silence gaps > 2 seconds)
- [x] Mood changes clear queue and load new tracks
- [x] Queue auto-refills when running low
